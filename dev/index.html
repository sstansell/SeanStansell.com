Vehicle Data Processing System

I built and maintaned a vehicle data import/export system for our automotive website platform. Loosely based on the Linux principle of chaining simple programs together to perform complex tasks, it was a highly modular system that handled thousands of imported text and image files and millions of records per day. It used a combination of .NET and PHP programs, communicating with MS SQL Server database and Microsoft BizTalk Server, to download, parse, clean, format, import, and append data to files from over 100 vendors. It then posted data updates to hundreds of websites based on the changes in the files. Subsequently it packaged data in dozens of different file formats and distributed them to hundreds of data partners. This system was in production for over a decade before the website platform it supported was retired. During its lifetime it provided the company a competitive edge, being able to work with any data partner in the industry, at no extra charge to the dealer.

SeanStansell.com

This site was built, in part, to prove to myself that I can still design and build a highly usable, fast website using modern tools and technologies, but rooted in solid principles and techniques. It's plain old HTML5 for the content, CSS compiled with SASS for styling, minimal Javascript where needed to enhance the presentation. The design is loosely based on a free template from a very talented designer, though highly modified for my purposes. It's setup using Gulp as a build system, automating the packaging, compression, minification, and distribution of site files.
